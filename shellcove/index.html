English

Your Mac, live in your hand

Shellcove turns your iPhone or iPad into a local Mac remote. Stream the desktop as video and drive it by touch: move the cursor, pinch to zoom, scroll, and drag. Compose text with the iOS keyboard, iOS Dictation, or any third-party input method, then send it to the focused Mac window. Add a full keyboard, mouse buttons, live Window Text, and clipboard sync for terminal work, AI coding agents, demos, and long-running tasks. Works over Wi-Fi, Tailscale, or any VPN that exposes both ends. TLS, no accounts, no cloud, no telemetry.

Live screen / 实时投屏
简体中文
  • 把 Mac 桌面以硬件视频流投到手机（ScreenCaptureKit 采集 + VideoToolbox 编码，iOS 硬件解码，低 CPU / 低功耗）—— 不是逐帧截图
  • 视频本身就是操控面：点按或拖动移动 Mac 光标、捏合放大精确定位、双指滚动、长按拖拽选择文本 / 移动窗口
  • 内置键盘按钮提供鼠标点击（单击 / 右键 / 双击 / 长按）和全部按键；多显示器可任选一块投屏
  • 远程桌面优先走 UDP/DTLS 媒体通道，不可用时回落到 TCP；弱网时优先保留清晰度，降低帧率和码率来恢复画面
English
  • Streams the Mac desktop as a hardware video feed (ScreenCaptureKit capture + VideoToolbox encode → hardware decode on iOS, low CPU/power) — not frame-by-frame screenshots
  • The video is the control surface: tap or drag to move the Mac cursor, pinch to zoom for precision, two-finger scroll, long-press to drag (select text, move windows)
  • A built-in keyboard button adds mouse clicks (Click / Right / Double / Long-press) and every key; pick any monitor to mirror on a multi-display Mac
  • Remote Desktop prefers a UDP/DTLS media channel and falls back to TCP when needed; on weak links it preserves clarity first, then lowers frame rate and bitrate so the picture recovers
Text input / 输入到 Mac
简体中文
  • 底部「输入到 Mac」打开全屏输入工作区，自动聚焦系统键盘；在远程桌面或窗口文本全屏里也能打开输入层，不压缩正在看的内容
  • 用户可以用普通键盘、iOS 听写、微信输入法或任意第三方输入法组合文本；Shellcove 接收最终文本并逐字符键入 Mac 当前焦点窗口
  • Shellcove 不请求麦克风或 Speech Recognition 权限。iOS 听写属于系统键盘能力，识别过程由系统输入法完成
  • 所有 app 都能用，包括 Ghostty、iTerm 等现代终端 —— 给 AI 编程 agent（Claude Code / Codex / Gemini CLI / aider / Cursor）发 prompt、确认长任务格外顺手
English
  • The bottom "Input to Mac" bar opens a full-screen input workspace and focuses the system keyboard. Inside Remote Desktop or Window Text, the input overlay opens without squeezing the content you are watching
  • Compose with the regular keyboard, iOS Dictation, WeChat Keyboard, or any third-party input method. Shellcove receives the final text and types it character by character into the focused Mac window
  • Shellcove does not request Microphone or Speech Recognition permission. iOS Dictation is a system keyboard feature handled before text reaches Shellcove
  • Works in every app, including modern terminals like Ghostty and iTerm. It is built for sending prompts and confirmations to AI coding agents such as Claude Code, Codex, Gemini CLI, aider, and Cursor

Clipboard + Window text / 剪贴板和窗口文本
简体中文
  • 剪贴板双向同步：Mac 剪贴板一变就自动镜像到已配对设备（写入 iOS 端时静默，无粘贴横幅）；点一下把设备剪贴板发回 Mac
  • 「窗口文本」工具：打开 Mac 当前焦点窗口文字的实时等宽视图（比如终端缓冲区），内容变化即刷新，宽行可横滑，不走视频管线
  • 文本通过 Accessibility 读取真实字符（无 OCR），保留终端原始换行，适合阅读 AI agent 输出、日志和长命令结果
English
  • Bidirectional clipboard sync: the Mac's clipboard mirrors to paired devices the moment it changes (set silently on iOS, no paste banner); send your device's clipboard back to the Mac with one tap
  • "Window Text" opens a live, monospaced view of the focused Mac window's text, such as a terminal buffer. It refreshes as content changes, supports wide-line scrolling, and does not use the video pipeline
  • Text is read as real characters via Accessibility, not OCR. It keeps terminal line breaks and is ideal for AI-agent output, logs, and long command results
Networking / 网络
简体中文
  • Mac 把可达的局域网 / Tailscale / VPN 地址写入配对二维码，iOS 以 endpoint 为单位保存，不会把 relay 主机和 Mac 本地端口混用
  • iOS 自动尝试可达 endpoint，也能在设置里手动添加中转机 host:port；每行显示的是 TCP 可达性/连接耗时，不是 ICMP ping
  • 直接连接支持 UDP/DTLS 远程桌面媒体通道；手动 relay endpoint 当前使用 TCP 视频回退路径，控制与输入仍然保持 TLS 加密
English
  • Mac writes reachable LAN / Tailscale / VPN addresses into the pairing QR. iOS stores endpoints as units, so a relay host is never accidentally combined with the Mac's local port
  • iOS tries reachable endpoints automatically and lets you add a manual relay host:port in Settings. Each row shows TCP reachability/connect time, not ICMP ping
  • Direct connections can use the UDP/DTLS media channel for Remote Desktop. Manual relay endpoints currently use the TCP video fallback while control and input remain protected by TLS
Privacy / 隐私
简体中文
  • 完全本地。Shellcove 只在你的 iPhone / iPad 和 Mac 之间通信，数据不会离开你的设备
  • TLS + 证书钉扎。Mac 首次启动生成自签证书，设备在配对时把指纹固定下来，只有你配对过的那台 Mac 能完成握手
  • 双向认证。配对二维码还带一枚每台 Mac 独有的 256 位密钥，设备每次连接都要出示；验证通过前 Mac 丢弃它的一切消息 —— 局域网里别的设备摸到端口也注入不了按键
  • 无账号，无服务器，无统计，无第三方 SDK，无后端
  • 文本由系统键盘或你选择的输入法生成；Shellcove 只接收最终文本，不请求麦克风或语音识别权限
English
  • 100% local. Shellcove communicates only between your iPhone / iPad and Mac. Nothing leaves your devices.
  • TLS with certificate pinning. Mac generates a self-signed cert at first launch; your device pins its fingerprint at pairing. Only your paired Mac can complete the TLS handshake.
  • Client authentication. The pairing QR also carries a 256-bit per-install secret the device presents on every connect; until it validates, the Mac drops all of that client's messages — so no other device that reaches the port can inject input.
  • No accounts, no servers, no telemetry, no third-party SDKs, no backend.
  • Text is composed by the system keyboard or input method you choose. Shellcove receives only the final text and does not request Microphone or Speech Recognition permission.

  2. iPhone 或 iPad 上点击「扫描二维码」
  3. 之后主页很干净：连接状态、「远程桌面」与「窗口文本」两个工具、当前 Mac 窗口、快捷按键、底部「输入到 Mac」。鼠标和完整键盘都在远程桌面里按需打开。
English
  1. Open Shellcove on your Mac — the main window shows a pairing QR
  2. On your iPhone or iPad, tap "Scan QR"
  3. The home is intentionally clean: connection status, Remote Desktop and Window Text tools, the current Mac window, shortcut keys, and the bottom Input to Mac bar. Mouse controls and the full keyboard open on demand inside Remote Desktop.
